# 出張申請おたすけ君

自家用車での出張距離を測定し、通勤経路と重複する区間を除算して、出張申請に必要な地図を印刷するためのWebアプリです。**Google CloudのAPIキー・課金設定は不要**です。

- 地図描画：OpenStreetMap（Leaflet）
- 住所・施設検索：**Googleジオコーダ（GAS経由）** ＋ 国土地理院 ＋ OpenStreetMap（Nominatim）を統合し、精度の高い順に表示
  - 施設名で良い結果が出ない（0件 or 市の中心など広域だけ）ときは、**OSMのPOI検索（Overpass API）** で周辺の施設を名前から補完します（無料・キー不要）。
  - ※ GASの無料Mapsサービスはジオコーダ（住所中心）まで。Googleマップアプリのような施設名検索（Places）はAPIキー＋課金が必要なため使えません。住所での検索がもっとも確実です。
- ルート検索：**Googleマップ（Google Apps Script の Maps サービス経由）**
  - フロント（HTML）から GAS のウェブアプリAPIを叩くハイブリッド構成。GASの内蔵Mapsサービスを使うため、Google CloudのAPIキーは不要です。
  - GASに接続できない場合は、直線距離×1.3の推定値に自動フォールバックします。

## 仕組み（なぜ重複区間を引くのか）
通勤手当は「自宅 ↔ 勤務地」の通勤経路に対して既に支給されています。
そのため出張のとき、出張経路のうち通勤経路と重なる区間は二重に支給されないよう距離から差し引きます。

**申請距離 ＝ 出張経路の総距離 − 通勤経路との重複区間**

このアプリは、出張経路と通勤経路（自宅↔勤務地）を実際の道のりで計算し、
重なる区間を自動検出して地図上に強調表示しながら、申請に出す距離を算出します。

## セットアップ（経路検索の有効化）
ルート検索に Googleマップを使うため、先に GAS（Google Apps Script）バックエンドをデプロイします。

1. [script.google.com](https://script.google.com) で新規プロジェクトを作成
2. リポジトリの `コード.gs` の内容を貼り付けて保存
3. 「デプロイ」→「新しいデプロイ」→ 種類「**ウェブアプリ**」
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
4. 発行された「ウェブアプリURL（`.../exec`）」をコピー
5. `index.html` 内の `const GAS_API_URL = '...'` にそのURLを貼り付け

> 未設定でもアプリは起動しますが、距離は直線距離×1.3の推定値になります。

## 使い方
ブラウザ（Edge / Chrome）で `index.html` を開くだけです。

1. **【最初の1回だけ】通勤ルートの確定**：自宅・勤務場所（学校）を登録し、「通勤ルート（初期設定）」でいつもの通勤ルートを1本確定します。提案ルートが実際の通勤路と違う場合は、「🖊 手動で調整」から地図をクリックして経由地を足し、実際に通る道に合わせて保存できます。確定した通勤ルートはブラウザに保存され、以降ずっと基準として使われます（自宅・学校を変えたときだけ再設定）。
2. **毎回の出張**：用務地を登録（**複数可**。上から回る順に並び、▲▼で並べ替え／「🔀最短の順に並べ替え」で自動最適化）→ 出張パターンを選択 → 距離を計算。確定済みの通勤ルートと重なる区間が自動検出され、その分を差し引いた申請距離が区間ごと（順路１・２・３…）に出ます → ルートを決定 → 地図を印刷。
   - 用件欄に貼る文章（例：`○○会議（順路１：1.5km、２：3.2km、３：4.0km） ※直行（自宅住所）`）も自動生成されます。

### 出張パターン
- ① 学校 → 用務地 → 学校（除算なし）
- ② 自宅 → 用務地 → 学校（行きの通勤経路と重複する区間を除算）
- ③ 学校 → 用務地 → 自宅（帰りの通勤経路と重複する区間を除算）
- ④ 自宅 → 用務地 → 自宅（行き帰りの通勤経路と重複する区間を除算）

## 公開方法

### A. GASでアプリごと配信（学校などGitHubが開けない環境向け・推奨）
学校PCは `script.google.com` だけ見られればよく、HTMLの取得はGAS（Googleのサーバー）が代行します。
**巨大HTMLを手で貼り付ける必要はありません。**
1. GitHubリポジトリを **Public** にする（GASがHTMLを読みに行くため）
2. GAS プロジェクトに `コード.gs` を貼り付け、`APP_HTML_URL` が自分のリポジトリの raw URL になっているか確認
   （例：`https://raw.githubusercontent.com/edupower07/maps/main/index.html`）
3. デプロイ（ウェブアプリ／実行=自分／アクセス=全員）。初回は UrlFetch の権限を許可
4. 発行された `.../exec` を開くと **アプリ画面が表示**される（GASが `index.html` を取得して配信）
5. Googleサイトに、その exec URL を iframe で埋め込む：
   ```html
   <iframe src="https://script.google.com/macros/s/XXXX/exec" width="100%" height="900" frameborder="0"></iframe>
   ```
   （Googleサイト → 挿入 → 埋め込む → 「コードを埋め込む」に貼り付け）

> `index.html` を更新すると、最大10分ほどで配信内容も自動更新されます（手貼り不要）。
> 印刷は埋め込み枠内だと不安定なので、印刷時は exec URL を直接開いて使ってください。

### B. GitHub Pages（リポジトリがPublicの場合）
**Settings → Pages** で Branch を `main` ／ フォルダを `/ (root)` にすると、
`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。

## フォルダ構成
- `index.html` … アプリ本体（GASが自動取得して配信／GitHub Pages・ローカルでも開ける）
- `出張申請おたすけ君.html` … 同じ本体（ローカルで開く用）
- `コード.gs` … アプリ配信＋経路/住所検索API（Google Apps Script にデプロイして使う）

## 注意
- 保存（自宅・勤務地・よく使う用務地）は、使っているブラウザ内に保存されます（端末間の自動同期はありません）。
- 住所検索・地図タイルは公開API（https）を利用します。ネットワークが `openstreetmap.org` / `unpkg.com` / `gsi.go.jp` への接続を許可している必要があります。
- ルート検索は GAS のウェブアプリ（`script.google.com` / `script.googleusercontent.com`）への接続を許可している必要があります。
- 重複区間は三角不等式やルート形状から推定するため、実際と多少の誤差が生じる場合があります。正確な値が必要なときは地図上で経路を確認してください。
